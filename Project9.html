Treasure Hunting Robot

The Brief

As the title describes, in a pair, we designed an X-Y gantry robot attachment that was to scan over an area for 5 magnets and then mark the locations of these magnets using plastic cups. I worked on the control system and search/ place algorithms, using MATLAB to control the robot (tethered system).

Challenges

Did I mention we only had 2 weeks to do this project from brief to final presentation (on top of other University course work)? This meant we could not make any design mistakes; redoing work would cost us dearly.

Development

Having a giant whiteboard next to my desk really helped with designing the searching and placing algorithms. I could map out search nodes and graphically express what I wanted the robot to do. A Hall-Effect sensor was used to detect magnets.

Final Design: Searching

Before searching could begin, the robot needed to know how big the beach was (it measured the scan area). The search algorithm was simple: it scanned over an area in a snaking pattern in 6 mm steps.

Final Design: Location Treasure

Locating peaks was done by building up a matrix “treasure map” of sensor data then looking for the largest magnetic peak. Once found, this peak (and local area around the peak) would be reduced to 0, allowing for the next highest peak to be found. Repeating 5 times gave the 5 treasure locations.

Final Design: Marking

Marking would calculate the distance to the nearest magnet, then move to it, place the marker and cross that location off the list of magnets to mark.

Skills

With the serious deadline in place, my coding under pressure was massively boosted. I also had to work closely with the hardware developer through the design-make-test cycle.
