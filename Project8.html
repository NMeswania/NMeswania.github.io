Educational Robot Scheme: UI

Project Brief

The Educational Robot Scheme was a semester long University project in a group of 6. We were tasked with creating a workshop for students to educate them on coding using the Dyson 360EYE robot. Our solution workshop included 5 challenges for the students to solve using a modified 360EYE: it could have either a gripper-arm or a ping-pong ball launcher. With these attachments, the students would create a script in Google Blockly that would control the robot to solve the challenge.

Other related sub-projects: Communication, Electronics Hardware

Sub-system Brief

Another part of my role, I created a Simulation UI for the students to simulate sections of code without connecting to the physical robot, saving time and improving their coding skills.

Challenges

We were targeting a few different age groups so the UI needed to be appropriate for each target group. Also, the UI couldn’t be part of the Blockly UI as the Blockly UI was part of the Dyson App which we didn’t have access to.

Development

Most great concepts come from breaking down a problem and brainstorming each topic. Whilst this may not be a “great” concept, this is how I tackled the problem: decomposition and brainstorming.

Final Design

There are two ways a students can observe the robot: a side-view for watching the robotic arm, and a plan-view for seeing the movement of the robot. Younger students could record some motion in the UI then compile that into code to understand the link between code and motion-output.

Further Work

The biggest step is to integrate the UI within the application and test/ debug all the issues that will undoubtedly arise.

Skills

My first time in making a UI; I had to keep the users in mind the entire time and present quite technical information in a very easy-to-understand way.
