Machine Vision Robot

Brief

As part of a group of 4 working on a rolling robot, my task was to design the robots’ triangulation algorithm in Python, using OpenCV with ArUco codes.

ArUco Codes

ArUco codes are markers, similar looking to QR codes, but for different applications. They are used to calculate the position of a camera relative to the code, often for projecting 3D images in Augmented Reality (that’s where the “Ar” in “ArUco” comes from). There’s a set of handy OpenCV library functions that deals with ArUco codes, making my job a little easier.

Triangulation

The real challenge here was to determine the robots’ position in a roughly 2m-square area with ArUco codes all around the grid, where the robot could calculate its position regardless of which code it was looking at. For this, I used the same principle used in robotics kinematics - Homogenous Transforms. It allowed me to calculate the robots’ position relative to the code, with the position of the code relative to the global coordinate frame, it was easy to then apply a Homogenous Transform to the first calculation to calculate the robot’s position in the global coordinate frame.

Communication and Signal Tidy-Up

Firstly, the algorithm was written in Python on the JeVois camera GPU, with the motor controllers written in Simulink on an Arduino MKR1000. So, I needed to write a Serial communication protocol between these two devices and decode the information (diagram here). Luckily the grid was sized where 1 bit corresponded to 1 cm and the whole X-Y coordinate system could be sent in 2 bytes - allows for faster communication. Finally, in Simulink, I removed some of the noise and used a Zero-Order hold before sending the data on to the motor controllers.

Further Work

Apart from increasing the positional resolution (by increasing the volume of information being sent), the only other improvement is the placement of the ArUco codes around the grid to maximise the coverage of the active area.

Skills

I really enjoyed working with the camera and being able to physically interact with the robot through the codes. The project lasted just 3 weeks, pretty intense considering we had other projects going on simultaneously. I want to improve my Python skills, seems like a really useful language to have in my toolbox.
